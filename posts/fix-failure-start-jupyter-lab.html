Yesterday, I encountered a minor issue while attempting to launch a Jupyter Lab notebook on a new cloud setup. The notebook wouldn’t initiate because it was unable to locate a component called ‘nbclassic’. Below, you’ll find the terminal output and the solution to this problem:

Fix it by installing nbclass

Personal note: As a frequent user of JupyterLab and the JupyterLab-vim extension, I’ve encountered numerous package incompatibility issues. These can be quite a hassle, disrupting the smooth operation of the environment. The most straightforward solution I’ve found is to pin and install specific versions of these packages that are known to work well together. For instance, I’ve had success with JupyterLab version 3.6.5 and JupyterLab-vim version 0.16.0.
